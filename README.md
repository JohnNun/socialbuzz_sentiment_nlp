This project explores the use of machine learning models to perform sentiment analysis on social media posts, the goal is to classify and predict the sentiment behind online posts. The dataset used for model training is, SocialBuzz Sentiment Analytics obtained from Kaggle datasets. This dataset contains over 700 posts with metadata such as hashtags, timestamps, and sentiments. Initial data exploration revealed significant class imbalance, with many sentiment categories containing a single row of data, posing a challenge for robust model learning and generalization. For data preparation steps I, standardize text and column formatting, and clean text data by lowercasing and stripping whitespaces. The dataset was split into training and test sets with stratification to preserve class distribution with the majority of stop words removed, except for words that could drastically change the sentiment like the word ‘not’. Prior to fine tuning, I trained two baseline models, Multinomial Naive Bayes and Logistic Regression, I chose these two models due to their efficiency and suitability for text-based, multi-class sentiment classification. In the end, Logistic Regression outperformed the alternative and was selected as the final model. For further fine tuning, I stemmed and tested various TF-IDF vectorizer settings, ultimately setting max features at 250 for best results. Despite the model tuning and data preparation, the model's performance was constrained by data limitations, achieving a precision of 0.12 and recall of 0.13, with roughly 20% accuracy in sentiment prediction on the test data leading to high rates of false positives and false negatives for its predictions. Overall, the model can discern sentiment patterns for the majority classes but struggles to recognize the pattern for the minority classes.

# Business Understanding
Social media has become prevalent in day-to-day life, through daily postings of one’s daily life, thoughts and opinions, as well as keeping others informed on current events. Behind every post, comment, like, and retweet there is sentiment that can be gathered as data that can be used for actionable insights as well as understanding the “why” someone feels the way they do. For this model, the goal is to train a model to be able to classify, learn and predict the sentiment behind a social media post for future sentiment analysis.

# Data Understanding
The data used for this project is the [SocialBuzz Sentiment Analytics](https://www.kaggle.com/datasets/eshummalik/socialbuzz-sentiment-analytics) dataset found in [Kaggle](https://www.kaggle.com/datasets). This dataset is comprised of a little over 700 social media posts, containing hashtags, time posted, number of times a post was liked or retweeted, as well as the sentiment behind the post. Upon some exploratory data analysis, an immediate downside to this data is its wide distribution as well as many sentiment categories only have 1 row of data to work with which will make it difficult if impossible for the model to properly learn from these rows. Since this data does have quite a few sentiment values with just 1 row of data it does cause a severe imbalance within the dataset.

## Data Preparation
Luckily, this data is fairly clean to begin with with no missing values, and most columns having the proper data type. To prepare the data for model training, I first drop the two “unnamed” columns as these appear to just be copies of the data’s row index. After dropping the two “unnamed” columns, I lower case all the column titles for personal preference, and ease of typing. To further prepare the data for model training I create a simple function that will take in a dataframe as well as a list of column titles and lowercase the text as well as remove any possible whitespaces accidentally input before and after the string of text making everything uniform and easier to work with. Once the data is cleaned I train test split the data with a .25 test data value and a stratify parameter to try and maintain the same class distribution as this data is heavily unbalanced and some categories only have less than 2 rows of data. My last step for data preparation was removing the majority of stop words from the text column. I excluded the word ‘not’ from the stop words list as this word can change the context from dislike to like, essentially training the model incorrectly.

# Modeling
For the modeling process I first trained two base models, MultinomialNB and Logistic Regression. I trained an MultinomialNB as a potential model as not only are these models highly efficient and fast, but work well for sentiment analysis with text data found in social media posts or even reviews, not to mention these models are effective for multi-class problems. I also trained a Logistic Regression as a potential model because lr models are an efficient, and interpretable algorithm for NLP classification tasks such as sentiment analysis. Lr models also handle high-dimensional, sparse data like TF-IDF vectors well. Ultimately the model chosen as the final model was Logistic Regression as this model performed the best overall.

For model training I first trained a model without stemming to see how the model would perform at an absolute base level. From there I created a function that would stem the words in the test column and tested the model to see if there was any change in score. For the TF-IDF vectorizer I trained all base models with 10 for max_features to see how the model performed with minimal data. For the final model I set TfidfVectorizer to have max features of 250 as anything past this number the model wouldn’t learn or generalize further from the data and return lower cross val scores. I also implemented stemming to the final model as doing helped the model generalize the data better giving improved results. For the Logistic Regression model, I set max iteration to 25 as that was the models peak learning point, anything past 25 would show no model improvement. All other parameters were left at the default parament as any other parameter changes appeared to affect the models learning capabilities in a negative way.

# Evaluation
With the given data used for this model and the way it is distributed, the model **performed as expected**.  As a good chunk of the data only has *a value of 1 for its sentiment** it does not give the model a good chance to **properly learn a pattern** from the data, especially so for the values moved into the test data during the train test split. Despite there not being a proper amount of data for each sentiment, for the sentiments that had multiple values the model was able to somewhat learn from these values. 

Overall, the model was still able to **predict text sentiment about 1/5 of the time** with the test data. The model’s **precision scored a .12, and a recall of .13** meaning as it is now the model has **high false positives as well as false negatives** because the model couldn’t properly learn from data especially from the rows that only had 1 row. 

## Limitations
* A limitation I experienced when working with this data is **sentiment distribution**. While some values did have a couple of rows of data to work with, **the majority of the data only had 1 row of data** and this alone doesn't really allow the model to properly learn from the data.
* A second limitation I would say is the **data quantity**; in total the **data has about 732 rows** of data and by using 25% of the data as testing data that leaves **about 549 rows of data to train the model**.

## Next Steps
* While logistical regression model was a good starting point, using deep learning models such as **Convolutional Neural Network (CNN) or BERT** can help give **better insight** and provide **better results for sentiment prediction**.
* If the current sentiment categories aren’t an absolute must, then another possible next step could be to take data we already have and **run it through a pre-trained sentiment analysis model** to either update the sentiment values maybe **causing some categories to have more than just 1 row of data**, or even **getting a simple positive, negative, and neutral output** and retrain my model with those categories for better results.
* Another possible next step if time allows would be to **tokenize the text as a bigram** as it is very possible the **sentiment or context of a post changed** due to the original tokenization.

# Conclusion
Despite the model's shortcomings, the model did **perform as expected** considering how the data is distributed with some of the **sentiment categories having only 1 row of data** to train the model with. Overall, the model was still able to **accurately predict text sentiment a little over 1/5 of the time** with the test data. For the model’s **precision it scored a .12, and a recall of .13** meaning as it is now the model does have **high false positives as well as false negatives**. As stated before, the model performing as it did was to be expected and with the model’s **macro average sitting at a .11 and weighted average at .16**, this further solidifies our expected outcome of the **modeling struggling to properly learn from the data as there just isn’t enough**.
